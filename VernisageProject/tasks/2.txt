Пишем простейшую CMS на js по управлению блогом
Как и с предыдущим заданием по слайдеру несколько уровней сложности

1) Статьи можно создавать, редактировать, удалять.
Список статей организовать в виде таблицы с ссылками на редакирование / удаление.
Форму создания / редактирования переиспользовать.
Сделать это в виде SPA без роутинга, с применением knockout.
Например форма может скрываться через data-bind, если isFormView = false на той же странице, где и список статей
Доступ через /Admin/Articles/Create и т.д
Admin - в данном случае это Area из asp.net

Также можно просматривать статьи. К статьям можно оставлять комментарии (имя, комментарий)
Их публичный просмотр добавить на сайт Вернисаж в меню

2) Добавить теги к статьям. По нажатию на тег статьи фильтруются по тегу.

3) Доступ к UI в пункте №1 сделать только для авторизованных пользователей.

4) Источником данных сделать серверное апи

5) Доступ к апи по удалению, редактированию, созданию статей сделать только для авторизованных пользователей.
